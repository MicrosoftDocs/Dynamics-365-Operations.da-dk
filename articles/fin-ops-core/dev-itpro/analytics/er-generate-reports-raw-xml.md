[!include[banner](../includes/banner.md)]

<span data-ttu-id="effda-104">Du kan bruge det nye **RAW XML**-formatelement til at designe ER-formater, der genererer udgående dokumenter i XML-format.</span><span class="sxs-lookup"><span data-stu-id="effda-104">You can use the new **RAW XML** format element to design Electronic reporting (ER) formats that generate outgoing documents in XML format.</span></span> <span data-ttu-id="effda-105">I nogle tilfælde kan du vælge at føje ubehandlede XML-data til disse rapporter af en eller flere af følgende årsager:</span><span class="sxs-lookup"><span data-stu-id="effda-105">In some cases, you might prefer to add raw XML data to these reports for one or more of the following reasons:</span></span>

- <span data-ttu-id="effda-106">Det er nemmere at bruge ubehandlede XML-filer til det oprindelige design og løbende vedligeholdelse af en rapport, fordi XML-strukturen automatisk kan genereres ved at udføre et kørselsudtryk.</span><span class="sxs-lookup"><span data-stu-id="effda-106">It's more convenient to use raw XML for the original design and ongoing maintenance of a report, because the XML structure can be automatically generated by executing a runtime expression.</span></span> <span data-ttu-id="effda-107">Derfor er det ikke nødvendigt at definere flere bindinger for flere formatelementer på designtidspunktet.</span><span class="sxs-lookup"><span data-stu-id="effda-107">Therefore, multiple bindings don't have to be determined for multiple format elements at design time.</span></span> <span data-ttu-id="effda-108">Det er muligt, når de datakilder, du bruger, indeholder oplysninger, der kan bruges til at oprette XML-elementer, mens rapporten genereres.</span><span class="sxs-lookup"><span data-stu-id="effda-108">It is possible when the data sources that you're using contain information that can be used to make XML elements while the report is generated.</span></span>
