## <a name="overview"></a><span data-ttu-id="448a0-110">Overblik</span><span class="sxs-lookup"><span data-stu-id="448a0-110">Overview</span></span>

    - <span data-ttu-id="448a0-118">o Mens enhedstest og komponenttest er på lavt niveau og kan være fuldt dataagnostiske (ikke afhængige af eksisterende datasæt), afhænger forretningscyklussen eller regressionsvalideringstestene af nogle eksisterende data.</span><span class="sxs-lookup"><span data-stu-id="448a0-118">o Where unit tests and component tests are low level and can fully be data agnostic (not dependent on existing dataset), the business cycle or regression validation tests are dependent on some existing data.</span></span> <span data-ttu-id="448a0-119">Disse data omfatter opsætning, konfigurationsindstillinger (parametre) og masterdata (debitor, kreditorer, varer osv.), men aldrig transaktionsdata.</span><span class="sxs-lookup"><span data-stu-id="448a0-119">This data includes setup, configuration settings (parameters), and master data (customer, vendors, items, etc.), but never transaction data.</span></span> <span data-ttu-id="448a0-120">Sørg for, at hvis en af disse ændres under testen, tilbageføres de som en del af den endelige test.</span><span class="sxs-lookup"><span data-stu-id="448a0-120">Make sure that during the test, if any of these are being changed, that they are reverted back as part of the final test.</span></span>
