# <a name="manage-standard-cost-updates"></a><span data-ttu-id="08a87-103">Administrere opdateringer af standardomkostninger</span><span class="sxs-lookup"><span data-stu-id="08a87-103">Manage standard cost updates</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="08a87-107">Med metoden med to versioner bruges én version, der indeholder posterne med de oprindelige omkostninger, og en anden version, der indeholder poster med alle omkostningsopdateringer.</span><span class="sxs-lookup"><span data-stu-id="08a87-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost updates.</span></span> <span data-ttu-id="08a87-108">En væsentlig fordel ved metoden med to versioner er den klare fremstilling og sporing af omkostningsopdateringer i en særskilt efterkalkulationsversion, uden at den oprindelige efterkalkulationsversion påvirkes.</span><span class="sxs-lookup"><span data-stu-id="08a87-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting the original costing version.</span></span> <span data-ttu-id="08a87-109">Metoden med to versioner kan bruges til at identificere flere trinvise opdateringer, hvor de enkelte trinvise opdateringer har en særskilt efterkalkulationsversion, der indeholder de trinvise omkostningsposter.</span><span class="sxs-lookup"><span data-stu-id="08a87-109">The two-version approach can be used to identify multiple incremental updates, where each incremental update has a separate costing version that contains the incremental cost records.</span></span>
