## <a name="work-templates"></a><span data-ttu-id="9cad9-108">Arbejdsskabeloner</span><span class="sxs-lookup"><span data-stu-id="9cad9-108">Work templates</span></span>

<span data-ttu-id="9cad9-111">Arbejdsskabeloner består af et hoved og tilhørende linjer.</span><span class="sxs-lookup"><span data-stu-id="9cad9-111">Work templates consists of a header and associated lines.</span></span> <span data-ttu-id="9cad9-112">Hver arbejdsskabelon gælder en bestemt *arbejdsordretype*.</span><span class="sxs-lookup"><span data-stu-id="9cad9-112">Each work template is for a specific *work order type*.</span></span> <span data-ttu-id="9cad9-113">Mange arbejdsordretyper er knyttet til kildedokumenter som f.eks. købs- eller salgsordrer.</span><span class="sxs-lookup"><span data-stu-id="9cad9-113">Many work order types are associated with source documents, such as purchase or sales orders.</span></span> <span data-ttu-id="9cad9-114">Andre arbejdsordretyper repræsenterer separate lagerstedsprocesser, f.eks cyklusoptælling.</span><span class="sxs-lookup"><span data-stu-id="9cad9-114">However, other work order types represent separate warehouse processes, such as cycle counting.</span></span> <span data-ttu-id="9cad9-115">*Arbejdspulje-id'et* gør det muligt at organisere arbejdet i grupper.</span><span class="sxs-lookup"><span data-stu-id="9cad9-115">The *work pool ID* lets you organize work into groups.</span></span>

<span data-ttu-id="9cad9-119">Arbejdslinjerne repræsenterer de fysiske opgaver, der kræves for at behandle arbejdet.</span><span class="sxs-lookup"><span data-stu-id="9cad9-119">The work lines represent the physical tasks that are required in order to process the work.</span></span> <span data-ttu-id="9cad9-120">For en udgående lagerproces kan der for eksempel være en arbejdslinje til plukning af varer inden for lagerstedet og en anden linje til placering af varerne i et stadieinddelt område.</span><span class="sxs-lookup"><span data-stu-id="9cad9-120">For example, for an outbound warehouse process, there might be a work line for picking up the items within the warehouse and another line for putting those items into a staging area.</span></span> <span data-ttu-id="9cad9-121">Der kan derefter være en ekstra linje til plukning af varer fra det stadieinddelte område og en anden linje til placering af varerne på en lastbil som en del af læsseprocessen.</span><span class="sxs-lookup"><span data-stu-id="9cad9-121">There can then be an additional line for picking the items from staging, and another line for putting the items into a truck as part of the loading process.</span></span> <span data-ttu-id="9cad9-122">Du kan angive en *direktivkode* på arbejdsskabelonlinjerne.</span><span class="sxs-lookup"><span data-stu-id="9cad9-122">You can set a *directive code* on work template lines.</span></span> <span data-ttu-id="9cad9-123">Et direktivkode er knyttet til et placeringsdirektiv og sikrer derfor, at lagerstedsarbejdet behandles på den korrekte placering på lagerstedet.</span><span class="sxs-lookup"><span data-stu-id="9cad9-123">A directive code is linked to a location directive and therefore helps guarantee that the warehouse work is processed in the correct location in the warehouse.</span></span> 

<span data-ttu-id="9cad9-124">Du kan oprette en forespørgsel til at styre, hvornår en bestemt arbejdsskabelon bruges.</span><span class="sxs-lookup"><span data-stu-id="9cad9-124">You can set up a query to control when a particular work template is used.</span></span> <span data-ttu-id="9cad9-125">Du kan for eksempel angive en begrænsning, så en bestemt skabelon kun kan bruges til arbejde på et bestemt lagersted.</span><span class="sxs-lookup"><span data-stu-id="9cad9-125">For example, you can set a limitation so that a particular template can be used for work only in a specific warehouse.</span></span> <span data-ttu-id="9cad9-126">Du kan også have flere skabeloner, der bruges til at oprette arbejde til behandling af udgående salgsordrer, afhængigt af den salgsoprindelsen.</span><span class="sxs-lookup"><span data-stu-id="9cad9-126">Alternatively, you might have several templates that are used to create work for outbound sales order processing, depending on the sales origin.</span></span> <span data-ttu-id="9cad9-127">Systemet bruger feltet **Løbenummer** til at bestemme den rækkefølge, som de tilgængelige arbejdsskabeloner vurderes i.</span><span class="sxs-lookup"><span data-stu-id="9cad9-127">The system uses the **Sequence number** field to determine the order that the available work templates are assessed in.</span></span> <span data-ttu-id="9cad9-128">Derfor, hvis du har en meget specifik forespørgsel til en bestemt arbejdsskabelon, skal du give den et lavt løbenummer.</span><span class="sxs-lookup"><span data-stu-id="9cad9-128">Therefore, if you have a very specific query for a particular work template, you should give it a low sequence number.</span></span> <span data-ttu-id="9cad9-129">Denne forespørgsel vil derefter blive evalueret før andre, mere generelle forespørgsler.</span><span class="sxs-lookup"><span data-stu-id="9cad9-129">That query will then be evaluated before other, more general queries.</span></span> 

<span data-ttu-id="9cad9-130">For at stoppe eller afbryde en arbejdsproces kan du bruge indstillingen **Stop arbejde** på arbejdslinjen.</span><span class="sxs-lookup"><span data-stu-id="9cad9-130">To stop or pause a work process, you can use the **Stop work** setting on the work line.</span></span> <span data-ttu-id="9cad9-131">I så fald bliver den arbejder, der udfører arbejdet, ikke bedt om at udføre det næste arbejdslinjetrin.</span><span class="sxs-lookup"><span data-stu-id="9cad9-131">In that case, the worker who is performing the work won't be asked to perform the next work line step.</span></span> <span data-ttu-id="9cad9-132">For at gå videre til næste trin skal den arbejder eller en anden arbejder vælge arbejdet igen.</span><span class="sxs-lookup"><span data-stu-id="9cad9-132">To move on to the next step, that worker or another worker must select the work again.</span></span> <span data-ttu-id="9cad9-133">Du kan også adskille opgaverne i et stykke arbejde ved hjælp af et andet *arbejdsklasse-id* på arbejdsskabelonlinjerne.</span><span class="sxs-lookup"><span data-stu-id="9cad9-133">You can also separate the tasks within a piece of work by using a different *work class ID* on the work template lines.</span></span>

## <a name="location-directives"></a><span data-ttu-id="9cad9-134">Lokationsvejledninger</span><span class="sxs-lookup"><span data-stu-id="9cad9-134">Location directives</span></span>
<span data-ttu-id="9cad9-135">Lokalitetsvejledninger er regler, der hjælper med at identificere pluk og læg-lokationer for lagerbevægelser.</span><span class="sxs-lookup"><span data-stu-id="9cad9-135">Location directives are rules that help identify pick and put locations for inventory movement.</span></span> <span data-ttu-id="9cad9-136">I f.eks. en salgsordretransaktion bestemmer en lokalitetsvejledning, hvor varerne plukkes, og hvor de plukkede varer skal lægges på lager.</span><span class="sxs-lookup"><span data-stu-id="9cad9-136">For example, in a sales order transaction, a location directive determines where the items will be picked, and where the picked items will be put.</span></span> <span data-ttu-id="9cad9-137">Lokationsvejledninger består af et hoved og tilhørende linjer, og du opretter dem på siden **Lokationsvejledninger**.</span><span class="sxs-lookup"><span data-stu-id="9cad9-137">Location directives consist of a header and associated lines, and you create them on the **Location directives** page.</span></span> 

<span data-ttu-id="9cad9-138">I hovedet skal hver lokationsvejledning være knyttet til en *arbejdsordretype*, der angiver den type lagertransaktion som direktivet anvendes til, f.eks. salgsordrer, genopfyldning eller plukning af råvarer.</span><span class="sxs-lookup"><span data-stu-id="9cad9-138">On the header, each location directive must be associated with a *work order type* that specifies the type of inventory transaction that the directive will be used for, such as sales orders, replenishment, or raw material picking.</span></span> <span data-ttu-id="9cad9-139">*Arbejdstype* angiver, om lokationsvejledningen bruges til at plukke eller placere arbejde eller til en anden lagerstedproces som f.eks. optælling eller ændringer i lagerstatus.</span><span class="sxs-lookup"><span data-stu-id="9cad9-139">The *work type* specifies whether the location directive will be used for picking or putting work, or for some other warehouse process, such as counting or inventory status changes.</span></span> <span data-ttu-id="9cad9-140">Du skal også angive et *sted* og et *lager*.</span><span class="sxs-lookup"><span data-stu-id="9cad9-140">You must also specify a *site* and a *warehouse*.</span></span> <span data-ttu-id="9cad9-141">En *lokationskode*, som du angiver i hovedet, kan bruges til at kæde lokationsvejledningen sammen med en eller flere arbejdsskabeloner.</span><span class="sxs-lookup"><span data-stu-id="9cad9-141">A *directive code* that you specify on the header can be used to link the location directive to one or more work templates.</span></span>
