### <a name="frequency-for-change-based-events"></a><span data-ttu-id="92d94-116">Frekvens for ændringsbaserede hændelser</span><span class="sxs-lookup"><span data-stu-id="92d94-116">Frequency for change-based events</span></span>

<span data-ttu-id="92d94-117">For ændringsbaserede hændelser kan du oprette et batchjob, der udløser afviklingen af en hændelse, lige efter at hændelse er logget af systemet.</span><span class="sxs-lookup"><span data-stu-id="92d94-117">For change-based events, you can set up a batch job that triggers the processing of an event soon after the event is logged by the system.</span></span> <span data-ttu-id="92d94-118">Hvis du konfigurerer, at batchjobbet skal gentages oftere, modtager brugerne deres påmindelser hurtigere, når der opstår ændringer.</span><span class="sxs-lookup"><span data-stu-id="92d94-118">If you set up the batch job to recur more often, users receive their alerts sooner after changes occur.</span></span> <span data-ttu-id="92d94-119">Men en høj frekvens til batchbehandling kan påvirke systemets ydeevne negativt.</span><span class="sxs-lookup"><span data-stu-id="92d94-119">However, a high frequency for batch processing might adversely affect system performance.</span></span>

<span data-ttu-id="92d94-120">På den anden side kan et batchjob, der skal gentages mindre hyppigt, og som der er planlagt til tidspunkter, hvor systembelastningen er lav, hjælpe med til at forbedre systemets ydeevne.</span><span class="sxs-lookup"><span data-stu-id="92d94-120">On the other hand, a batch job that recurs less often, and that is scheduled for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="92d94-121">En lav frekvens for batchbehandling kan imidlertid ikke opfylde brugernes behov for rettidig beskeder.</span><span class="sxs-lookup"><span data-stu-id="92d94-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="92d94-122">Så når frekvensen for de ændringsbaserede hændelsesbatches angives, skal du være opmærksom på balancen mellem det tidsmæssige behov for påmindelser og ydeevnen for hele systemet.</span><span class="sxs-lookup"><span data-stu-id="92d94-122">Therefore, when you set up the frequency of batch processing for change-based events, consider the balance between the timeliness of alerts and the performance of the whole system.</span></span> <span data-ttu-id="92d94-123">Jo flere brugere, der opretter påmindelsesregler, desto mere relevant er disse overvejelser.</span><span class="sxs-lookup"><span data-stu-id="92d94-123">These considerations become more relevant as the number of users who create alert rules increases.</span></span> <span data-ttu-id="92d94-124">Hyppigheden påvirker ikke antallet af hændelser, der skal behandles.</span><span class="sxs-lookup"><span data-stu-id="92d94-124">The frequency doesn't affect the number of events that must be processed.</span></span> <span data-ttu-id="92d94-125">Men hvis flere brugere opretter regler, skal der udføres flere kontroller.</span><span class="sxs-lookup"><span data-stu-id="92d94-125">However, if more users create rules, more checks must be done.</span></span> <span data-ttu-id="92d94-126">Denne type af dataudveksling kan have indflydelse på systemets ydeevne.</span><span class="sxs-lookup"><span data-stu-id="92d94-126">This type of data exchange might affect system performance.</span></span>

#### <a name="the-risks-of-low-batch-frequency"></a><span data-ttu-id="92d94-127">Risiciene ved en lav batchfrekvens</span><span class="sxs-lookup"><span data-stu-id="92d94-127">The risks of low batch frequency</span></span>

<span data-ttu-id="92d94-128">Hvis du har angivet en lav frekvens for batchbehandling af ændringsbaserede hændelser, kan data, der er relevante for betingelserne for påmindelsesregler, muligvis være ændret før afviklingen af batchen.</span><span class="sxs-lookup"><span data-stu-id="92d94-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might be changed before the batch is processed.</span></span> <span data-ttu-id="92d94-129">Derfor kan du miste påmindelser.</span><span class="sxs-lookup"><span data-stu-id="92d94-129">Therefore, you might lose alerts.</span></span>

<span data-ttu-id="92d94-130">Der er f.eks. konfigureret en påmindelsesregel, der skal udløse en påmindelse, når hændelsen er **ændringer for kundekontakt**, og betingelsen er **kunde = BB**.</span><span class="sxs-lookup"><span data-stu-id="92d94-130">For example, an alert rule is set up to trigger an alert when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="92d94-131">Når der opstår ændringer for kundekontakten for kunde BB, logføres hændelsen med andre ord.</span><span class="sxs-lookup"><span data-stu-id="92d94-131">In other words, when the customer contact for customer BB is changed, the event is logged.</span></span> <span data-ttu-id="92d94-132">Batchafviklingssystemet er dog konfigureret sådan, at batchafvikling optræder mindre hyppigt end dataindtastning.</span><span class="sxs-lookup"><span data-stu-id="92d94-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="92d94-133">Hvis kundenavnet ændres fra **BB** til **AA**, før hændelsen er behandlet, svarer dataene i databasen ikke længere til betingelsen i reglen **kunde = BB**.</span><span class="sxs-lookup"><span data-stu-id="92d94-133">If the customer name is changed from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="92d94-134">Derfor genereres der ingen påmindelse, når hændelsen er endelig afvikles.</span><span class="sxs-lookup"><span data-stu-id="92d94-134">Therefore, when the event is finally processed, no alert is generated.</span></span>
